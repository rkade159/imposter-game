# Imposter Game

A web app version of the party game **Imposter** (pass-and-play, similar to Spyfall). Players pass a single device around — most see a secret word, while one or more imposters instead get a deliberately vague hint. Group discussion follows; the goal is to identify the imposter(s) without giving the word away.

## Status

**Playable start to finish** — a full round now runs Setup → Reveal → Pass → Discussion → **Results** → back to Setup. On Setup the host picks the player and imposter counts, the word source, and can optionally **type a name for each player** (the list of name fields grows/shrinks with the player count; blank fields fall back to "Player N"). On Start, roles are assigned with a Fisher–Yates shuffle and the device passes player-to-player: each player privately reveals whether they hold the secret word or are the imposter — using the **reveal animation chosen in Settings** (see below) — imposters are shown a deliberately vague hint to help them blend in, then the device hands off through a neutral pass screen. The reveal, pass, and results screens all use the typed **names** (e.g. "Pass to Sam", "The imposter was Priya") in place of "Player N". After everyone has revealed, the discussion screen offers a **"Reveal the imposter(s)"** button (behind a confirmation guard); confirming opens the **Results** screen, which names the imposter(s) and reveals both the secret word and the imposter's hint. **"Play again"** returns to Setup with the previous round's settings **and names** pre-filled. **Screen routing** is a `screen` field on `gameState` driving an `{#if}` ladder in `App.svelte`. A **Settings screen** (reached from a Settings button on Setup, shown in place of the form so the half-filled setup survives) now hosts app preferences: **Grayscale mode**, which turns the whole app black-and-white so the imposter's red reveal card can't be read by glance or reflection, the **Anti-Yusuf Feature**, a gag toggle that refuses to start the round and instead calls out whoever is last to pass the phone, and a **Reveal animation** selector that switches how each player learns their role between **Classic** (the quick tap-to-reveal), **Secret letter** (a sealed envelope you press and hold to open, the flap swinging up as a note slides out), **Wheel of Fate** (a rigged spinning wheel you hold to slow to a stop on your role), **Choose a Card** (a 3×3 grid of face-down cards — tap any one and it flips to your role while the other eight flip to decoy roles for show) and **Peek under** (a plain cover over your role that you press and swipe up to lift like a lid — it rises with your finger so you can glance underneath and drops straight back down when you let go; once you've peeked, your full role stays on screen with the advance button) — the choice persists across rounds and defaults to Classic. A **Reveal fellow imposters** toggle (off by default, and disabled unless the round has 2+ imposters) additionally shows each imposter the names of the *other* imposters during their reveal — across all five reveal styles — so a multi-imposter team can actually coordinate; with it off, an imposter only ever learns that they themselves are an imposter. A **Troll Mode** selector (off by default) adds an occasional chaos round in which **everyone** is secretly told they are the imposter — each player gets their own random hint and so thinks they alone are the imposter — chosen between three behaviours: **Sneaky** (a rare flat chance, never twice in a row), **Building** (odds that climb each round until they fire) and **Guaranteed** (the very next round, after which it switches itself back off); the surprise never lands on the round you enable it, the results screen stays ambiguous, and the cross-round odds persist across reloads. The word source picker now also offers a **Custom List** option: choosing it opens a builder (in place of the setup form) where the host searches the Common Nouns deck and taps words to hand-pick exactly which ones can be the secret word this round — only the words are shown (never the imposter's hint), an inline "Word not available." appears when a search matches nothing, and an always-present Confirm (disabled until at least one word is picked) locks the subset while a Back button exits without committing; the custom list is **round-scoped on purpose** — it is never saved, so "Play again" reverts the source to Common Nouns. A new **Roles screen** (reached from a **Roles** button in the top-left of Setup, opposite Settings) hosts an optional third role, the **Jester** (off by default, persisted): when on, one player per round is secretly the jester — they're given the **real word** like a crewmate but win by **getting voted out on purpose**, so they try to act like the imposter. The jester is shown in **light pink** across all five reveal animations (and neutralises under Grayscale like the other roles), a banner announces that "a jester is among you" during the reveal and discussion phases, and the **Results** screen names who the jester was; the win is settled **verbally at the table** (the app has no in-app voting). The jester needs **3+ players** (1 jester + ≥1 imposter + ≥1 crewmate) and reserves a slot so the imposter count caps at *players − 2*; a **Troll Mode** round overrides it (everyone's the imposter, no jester that round). A **second** optional role, the **Prosecutor** (also on the Roles screen, off by default and persisted), turns one of the round's imposters into a secret assassin: they don't know the word (they get the imposter's hint like any imposter) but are privately told **one specific player to get voted out** — pull it off and they win the round instantly. The target is always a crewmate-type (a plain crewmate or the jester — never a fellow imposter and never themselves), so if their mark happens to be the jester and the table votes them out, **both win**. The Prosecutor shows in **gold** across all five reveal animations (and neutralises under Grayscale), counts as an imposter for the *Reveal fellow imposters* list (shown there by name only, so the specific role never leaks), and is named — alongside their target — on the **Results** screen; like the jester there's **no in-app voting**, so the win is settled verbally, and it's skipped on a Troll Mode round. It **occupies an existing imposter slot** (it doesn't add one — so the imposter count is unchanged), needs **3+ players**, and stays **hidden** during the round (no banner — the reveal is the surprise). A **third** optional role, the **Lawyer** (also on the Roles screen, off by default and persisted), is the **mirror of the Prosecutor**: one crewmate is secretly made the lawyer — they get the **real word** like any crewmate, but are privately assigned a single **"client"** to defend, drawn at random from the round's imposter-aligned players (any imposter **or** the Prosecutor), and they win **with the imposters** if their client is **not** voted out. Crucially they're **never told the client is an imposter** — only to keep that player from being voted out — so the twist is theirs to work out. The lawyer shows as a normal crewmate reveal with an added **teal** note naming their client (across all five reveal animations, and neutralising under Grayscale), takes up an existing **crewmate** slot (so the imposter count is unchanged), needs **3+ players**, stays **hidden** during the round (no banner), is skipped on a Troll Mode round, and is named — alongside their client — on the **Results** screen; like the other roles there's **no in-app voting**, so the win is settled verbally at the table. The Prosecutor change **polishes the Wheel of Fate**: it settles to a stop faster, **snaps** straight to your role if you release in the last sliver of the spin (instead of speeding back up), shows a **loading bar** while you hold, and grows light-pink **jester wedges** on jester rounds. The **Results** screen now has its own optional animation, chosen by a separate **Imposter reveal** setting (default **Static** — the existing plain-text reveal): turning on **Spotlight** plays a dramatic end-of-round lead-in where the screen goes black, every player's name is scattered around it (white = crewmate, red = imposter, light-pink = jester), and a spotlight roams a random-looking path past the crewmate names — deliberately avoiding the red ones — before stopping on an imposter; with 2+ imposters the remaining imposters then **flare red together**, after which the usual results text (imposter, jester, word and hint) appears. It's purely theatre on top of the existing results — driven only by the round's roles, so it always stops on a real imposter (true even under Grayscale, where the colour tell is gone), with a **Skip** button + Enter/Space shortcut and a reduced-motion fallback that goes straight to the lit screen. Production build is clean. The app is also now **wrapped with Capacitor for Android** — the native project is generated and the build is ready to compile into an `.aab` and submit to the Google Play Store (web behaviour is unchanged; see the [runbook](02-development/workflow/03-builds/imposter-game-app/RUNBOOK.md)). Next up: real discussion mechanics (timer + voting), more Settings options (e.g. toggling the reveal confirmation), design polish for the setup screen's names section, and replacing the placeholder app icon/splash with designed artwork.

## Tech Stack

- **Svelte** + **Vite** + plain JavaScript
- **PWA** from day one (installable, offline-capable)
- Statically hosted on **Netlify** (git-connected auto-deploy from `main`)
- **Capacitor** wraps the app for **Android** (Google Play) — done. iOS deferred (needs a Mac); the codebase stays compatible for later.

Full reasoning in [`01-plan/plans/tech-stack-plan-final.md`](01-plan/plans/tech-stack-plan-final.md).

## Repo Layout

This repo isn't a typical app codebase — it's a **workspace system** with three siloed pipelines (planning, development, design). Each silo owns one stage of the work and routes agents in via its own `CONTEXT.md`.

```
imposter-game/
├── CLAUDE.md          ← Always-loaded map of the repo
├── CONTEXT.md         ← Task router (which workspace to go to)
├── CHECKLIST.md       ← Per-session reminders
├── README.md          ← You are here
│
├── 01-plan/           ← Brainstorming → polished plans
├── 02-development/    ← Plans → briefs → specs → built code
└── 03-design/         ← Plans → briefs → specs → built designs
```

Each workspace has its own `CONTEXT.md` with full per-task load tables — read those before working in a workspace.

## Working in This Repo

Start at [`CONTEXT.md`](CONTEXT.md) — it routes you to the right workspace for your task. The naming and file-placement conventions are documented in [`CLAUDE.md`](CLAUDE.md).

## Running the App

The scaffold lives at [`02-development/workflow/03-builds/imposter-game-app/`](02-development/workflow/03-builds/imposter-game-app/). From inside that folder:

- `npm install` — install dependencies (one time)
- `npm run dev` — local dev server with hot reload
- `npm run build` — production build to `dist/`
- `npm run preview` — serve the production build locally (use this to test PWA install)
- `npm run android:open` — build, sync, and open the Android project in Android Studio

Full notes and folder map are in the scaffold's [README](02-development/workflow/03-builds/imposter-game-app/README.md). Android build, signing, and Play Store submission steps are in the [RUNBOOK](02-development/workflow/03-builds/imposter-game-app/RUNBOOK.md).

## Long-Term Direction

Web app first, then native. The web version ships on Netlify and is now also wrapped with Capacitor for **Android** (Google Play) from the same codebase. **iOS** is deferred until a Mac is available — adding it is one command (`npx cap add ios`) on the existing project, no rewrite.
